# 文件传输客户端 - 图形界面版本

## 🎨 特点

- ✅ 简单易用的图形界面
- ✅ 无需记忆命令行参数
- ✅ 支持直连和代理连接
- ✅ 可视化操作日志
- ✅ 拖拽式文件/文件夹选择
- ✅ 实时连接状态显示

## 🚀 使用方法

### Windows 用户
双击运行 `启动GUI.bat` 即可

### 或直接使用 Python
```bash
python file_transfer_gui.py
```

## 📋 功能说明

### 1. 连接配置
- **目标服务器**: 输入服务器IP地址和端口
- **使用代理**: 勾选后可配置代理服务器
- **连接按钮**: 点击连接/断开服务器

### 2. 文件操作
连接成功后可以进行以下操作：

- **📤 上传文件**: 选择单个文件上传到服务器
- **📁 上传文件夹**: 上传整个文件夹（保持目录结构）
- **📋 列出文件**: 查看服务器上的所有文件
- **📥 下载文件**: 从服务器下载文件到本地

### 3. 操作日志
- 实时显示所有操作的详细信息
- 颜色标记：
  - 🟢 绿色 = 成功
  - 🔴 红色 = 错误
  - 🔵 蓝色 = 信息
  - 🟠 橙色 = 警告

## 💡 使用示例

### 直接连接
1. 输入目标服务器地址（例如：192.168.1.100）
2. 输入端口（默认：8080）
3. 点击"连接"按钮

### 通过代理连接
1. 勾选"使用代理"
2. 输入代理服务器地址和端口
3. 输入目标服务器地址和端口
4. 点击"连接"按钮

### 上传文件
1. 连接成功后，点击"上传文件"
2. 选择要上传的文件
3. 等待上传完成

### 上传文件夹
1. 点击"上传文件夹"
2. 选择要上传的文件夹
3. 确认上传
4. 所有文件会保持原有目录结构上传

## 🔧 技术说明

- 基于 `tkinter` 标准库（无需额外安装）
- 多线程处理网络操作，界面不会卡顿
- 自动处理文件路径和编码问题
- 兼容 Windows/Linux/macOS

## 📝 注意事项

- 需要 Python 3.6 或更高版本
- 确保 `file_transfer_client.py` 在同一目录
- 上传大文件时请耐心等待
- 断开连接前建议等待当前操作完成

## 🐛 常见问题

**Q: 提示找不到 file_transfer_client 模块？**  
A: 确保 file_transfer_client.py 和 file_transfer_gui.py 在同一目录下

**Q: 界面显示乱码？**  
A: Windows用户使用提供的 .bat 文件启动，已设置UTF-8编码

**Q: 上传文件夹后服务器看不到文件？**  
A: 检查服务器的uploads目录和权限设置

**Q: 连接后无响应？**  
A: 检查服务器是否正在运行，防火墙设置是否正确
